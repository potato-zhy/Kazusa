# Git Workflow（中文版）

## 用途

这份文件定义了 `Kazusa` 项目在多个 AI 并行参与时的最小 Git 工作流。

## 分支模型

- `main` 是集成分支。
- 高优先级的架构工作，只应在本地验证后进入 `main`。
- 次级或探索性工作应使用短生命周期的 feature branch。

## 并行工作规则

- 不要让两个 AI 在同一分支上同时进行无关修改。
- 如果需要并行工作，使用独立分支或独立 worktree。
- 合回 `main` 之前，先检查是否存在架构冲突。

## 提交规则

- 保持提交小而单一。
- 尽量做到每个提交只对应一个被解决的设计或实现步骤。
- 不要把宪法级设计变更和无关的 prototype 改动混在同一个提交里。

## 优先级规则

- 当并行工作流发生冲突时，高优先级的架构主线优先。
- 低优先级工作应 rebase 或适配，而不是强行让主线围绕它弯曲。

## 当前建议

- 把当前 continuity kernel 线路视为主线。
- 在 continuity kernel 还未稳定前，不要让 runtime stewardship 工作进入 `main`。
