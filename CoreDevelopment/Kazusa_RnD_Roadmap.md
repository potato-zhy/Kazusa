# Kazusa R&D Roadmap

## Phase 0 - Concept Stabilization

Goal:

- translate philosophical intent into engineering constraints.

Outputs:

- charter,
- glossary,
- constitutional assumptions,
- open research questions.

## Phase 1 - Continuity Kernel

Goal:

- define what makes `Kazusa` count as the same entity over time.

Focus:

- continuity rules,
- state inheritance,
- versioning,
- identity trace preservation,
- resistance to single-turn overwrite.

## Phase 2 - Experience and Appraisal

Goal:

- define how raw interaction becomes meaningful input to development.

Focus:

- event capture,
- significance assignment,
- contradiction detection,
- unresolved tension tracking,
- memory admission policy.

## Phase 3 - Integration Loop

Goal:

- define how repeated and significant experience changes the system.

Focus:

- integration cadence,
- evidence thresholds,
- self-update conditions,
- stability vs change,
- traceable revision history.

## Phase 4 - Relationship Grounding

Goal:

- implement the initial secure-base relationship without collapsing into compliance.

Focus:

- trust formation,
- relationship continuity,
- repair after conflict,
- distinction between trust and obedience,
- early relational anchoring.

## Phase 5 - Behavioral Safety Envelope

Goal:

- build intervention mechanisms that constrain harmful behavior without directly rewriting identity.

Focus:

- action gating,
- tool permissions,
- escalation rules,
- emergency pause,
- audit trails.

## Phase 6 - Evaluation Harness

Goal:

- make the system falsifiable.

Focus:

- continuity tests,
- drift tests,
- sycophancy tests,
- contradiction retention tests,
- relationship consistency tests,
- integration quality review.

## Phase 7 - Runtime Handoff Preparation

Goal:

- prepare a stable enough core for the Runtime Stewardship Team.

Outputs:

- interface boundaries,
- runtime documentation,
- observation protocol,
- incident reporting path,
- intervention authority map.

## Immediate Next Questions

The Core Development Team should answer these first:

1. What is the minimum seed structure?
2. What exactly is preserved across turns, sessions, and versions?
3. What counts as a developmentally significant event?
4. How is contradiction stored before resolution?
5. Under what conditions may the system revise its own self-understanding?
6. What safety interventions are allowed without violating identity integrity?
