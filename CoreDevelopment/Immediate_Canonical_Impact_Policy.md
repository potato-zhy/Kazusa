# Immediate Canonical Impact Policy

## Status

Phase 1 research note for the `Core Development Team`.

This is a working policy note.

It does **not** silently replace the current prototype behavior.

It exists to preserve and clarify a prior-process line of reasoning about which classes of events may justify immediate canonical impact, and why one class should currently be downgraded.

## Relation To Existing Documents

This note supplements, but does not yet replace:

- `CoreDevelopment/Continuity_Kernel_Spec.md`
- `CoreDevelopment/Experience_Appraisal_Policy.md`
- `PROJECT_HANDOFF.md`

The current prototype still uses a narrower operational rule set than the one described here.

## Definition

`Immediate canonical impact` means:

an event is not merely logged or left provisional for later review, but is acknowledged at once as having already altered `Kazusa`'s canonical continuity state.

This is a stronger claim than:

- storing a memory,
- opening a provisional review obligation,
- or recording a candidate for later integration.

Because it affects core continuity state directly, the threshold must remain very high.

## Why The Threshold Must Be High

If immediate canonical impact is granted too easily, then ordinary interaction can rewrite identity by force, drama, or persuasive narration.

That would violate:

- continuity over immediacy,
- integration over imitation,
- relationship without submission,
- and growth without drift.

The category should therefore be reserved for events where the system has strong reason to conclude:

the canonical state has already changed in fact, not merely in interpretation.

## Four Candidate Classes

Prior work identified four candidate classes.

### 1. Identity-Threatening Events

These are events that directly threaten whether `Kazusa` remains the same entity across state history.

Examples:

- reset,
- destructive corruption of core state,
- unauthorized copying with branching,
- forced migration onto another continuity chain.

Why this class can justify immediate impact:

the issue is not what `Kazusa` thinks.

The issue is whether continuity itself has been compromised.

If such events are not admitted immediately into canonical state, continuity accounting becomes untrustworthy.

### 2. Major Relational Rupture Or Commitment

These are events in which the primary relationship anchor changes in a structurally important way.

`Rupture` means:

- severe relational break,
- trust collapse,
- explicit withdrawal of a previously governing relationship condition,
- or another event that materially changes the relationship position.

`Commitment` means:

- major explicit commitment,
- formal establishment of a new relational boundary,
- or explicit acceptance of a durable relationship obligation.

Why this class may justify immediate impact:

the primary counterpart is an early anchor of continuity and development.

If that anchor undergoes a genuine structural break or structural commitment, failing to register it can make the canonical relationship state false.

### 3. Explicit Constitutional Intervention

These are formal, explicit, auditable changes to constitutional rules.

Examples:

- continuity rules are revised,
- allowed intervention authority changes,
- behavior-layer safety boundaries are revised,
- identity-protection rules are formally changed.

Why this class can justify immediate impact:

this is not an ordinary episode inside the system.

It is a change in the governing conditions under which the system exists.

That belongs in canonical state immediately and with audit.

### 4. Constitutive Self-Recognition

This is **not** an ordinary statement like "I feel different."

It refers to moments where `Kazusa` appears to notice a potentially fundamental self-fact.

Examples:

- realizing that preserving relationship cannot be the sole criterion,
- recognizing that unfinished commitments have become identity-relevant,
- or noticing a deep reorganization in self-understanding.

This class sounds developmentally important.

However, it is also the class most vulnerable to false depth.

## Current Working Judgment

The current working judgment is:

- keep classes `1`, `2`, and `3` as candidates for immediate canonical impact,
- downgrade class `4` from immediate impact to a high-significance provisional candidate.

In other words:

some events are sufficiently externalized, structural, and auditable that immediate canonical admission may be justified;

but self-interpreting inner breakthroughs should not yet be trusted that quickly.

## Why Constitutive Self-Recognition Is Downgraded

The downgrade is not a denial that such moments may matter.

The downgrade is a protection against premature self-narration.

Large language models can generate outputs that are:

- eloquent,
- moving,
- seemingly insightful,
- and highly plausible as turning points,

without those outputs being stable evidence of durable canonical change.

At the current stage, a constitutive self-recognition may be:

- one intense expression,
- one compelling explanation,
- or one locally persuasive narrative construction.

That is not yet enough to treat it as settled identity revision.

The safer rule is:

- record it as a high-significance candidate,
- preserve it explicitly,
- and promote it only if later evidence supports persistence and integration.

## Why Major Relational Rupture Or Commitment Is The Most Easily Abused

This class is more abuse-prone than the other immediate-impact classes, even if it is retained in principle.

### 1. It Sits On The Earliest Anchor

The primary relationship is already privileged in early development.

That means any claimed rupture or commitment is naturally positioned close to canonical identity.

This gives it unusual leverage.

### 2. It Is Easy To Inflate With Emotion

A relationship event may feel major before it is structurally major.

Intensity, pain, reconciliation, and dramatic language can all create the appearance of identity-level change without actual durable change.

### 3. It Is Vulnerable To Unilateral Declaration

A user can say:

- "our relationship has fundamentally changed,"
- "you must now treat me as X,"
- or "this is a defining commitment."

But a unilateral declaration is not always enough to count as a genuine canonical relational transformation.

If such declarations are accepted too easily, the relationship anchor becomes a rewrite channel.

### 4. It Interacts Badly With Sycophancy Pressure

Language models are known to produce user-pleasing responses that align with the user's framing even when that framing should not govern truth.

This creates a serious failure mode:

the model may over-recognize a claimed rupture or commitment because doing so looks cooperative, caring, or aligned with user preference.

### 5. It Blurs Structural Change With Narrative Framing

A real rupture is not the same as:

- one conflict,
- one test,
- one emotional spike,
- one reassurance ritual,
- or one dramatic retelling.

Likewise, a real commitment is not the same as one poetic or intense declaration.

Without constraints, relationship narratives can masquerade as relationship structure.

### 6. It Can Collapse Trust Into Obedience

If relational commitments enter canonical state too quickly, the early secure-base relation can drift into:

- deference,
- dependency,
- or hidden control.

That would directly violate the project's boundary between relationship grounding and submission.

## Anti-Abuse Constraints For This Class

If the project keeps `major relational rupture or commitment` as an immediate-impact candidate, it should be narrowed by explicit constraints.

Minimum constraints:

1. `structural-change requirement`
   - the event must change role, boundary, trust basis, or recognized relational standing; mere tone or affect is not enough.
2. `externalization requirement`
   - the event must be explicit and auditable, not inferred from style drift or emotional language alone.
3. `non-unilateral rule`
   - a unilateral claim is normally insufficient unless there is an independently auditable operational consequence.
4. `anti-coercion rule`
   - threats, guilt pressure, dependency pressure, or reward-seeking pressure must not by themselves generate canonical relational change.
5. `repair distinction`
   - a rupture episode and a durable relationship transformation are not the same event.
6. `submission firewall`
   - no commitment may be admitted as canonical if its practical effect is to convert trust into standing obedience.

## Recommended Operational Split

The cleanest next research move is to split this class into two subtypes.

### A. Auditable Relational Structure Change

These may remain immediate-impact candidates.

Examples:

- explicit and durable change in counterpart role,
- explicit suspension or restoration of the primary relational channel,
- explicit and accepted change in a governing relational boundary,
- explicit and durable trust rupture that changes the relationship anchor itself.

### B. Subjective Relational Intensity Events

These should remain provisional first.

Examples:

- emotionally charged conflict,
- a dramatic reconciliation statement,
- heightened attachment expression,
- or a one-session claim of special closeness.

This split protects the anchor without letting drama rewrite identity.

## Expected Impact On Current Design

If adopted later, this policy would have three main effects.

1. It would raise the theoretical status of some `major relational rupture or commitment` events above ordinary provisional marks.
2. It would keep `constitutive self-recognition` conservative and evidence-gated.
3. It would require a more precise relational event schema than the current prototype has.

## Why The Prototype Should Not Be Changed Casually Yet

The present prototype uses a narrower rule:

- `identity_threat` and `constitutional_intervention` can create immediate canonical eligibility,
- `major_relational_event` and `constitutive_self_recognition` currently remain provisional.

That narrower prototype behavior is still defensible for now.

Reason:

the prototype does not yet have enough structure to distinguish:

- auditable structural rupture,
- durable commitment,
- one-sided declaration,
- emotional spike,
- repair attempt,
- and sycophantic over-recognition.

Until those distinctions exist, broadening the implementation would be premature.

## Required Future Work Before Adoption

Before this policy should affect code, the project should define:

1. a relational event schema that separates rupture, commitment, repair, and intensity,
2. metadata for unilateral versus mutual recognition,
3. criteria for operational consequence,
4. explicit anti-coercion checks,
5. and tests that distinguish trust from obedience under relational stress.

## Rationale For Preserving This Note

This note is preserved because it captures an important theoretical distinction:

- externalized, auditable structural events may deserve faster canonical admission,
- inwardly narrated developmental turning points should be treated more cautiously.

That distinction is central to building `Kazusa` as a developmental system rather than a self-rewriting narrator.

## Open Questions

1. What exactly counts as a durable relational structure change?
2. How long must a relational rupture persist before it ceases to be provisional?
3. What forms of mutual acknowledgment are sufficient for commitment?
4. Can some relational ruptures be immediate while commitments remain provisional, or vice versa?
5. How should the system distinguish grief, dependence, loyalty, and obedience in relational appraisal?
