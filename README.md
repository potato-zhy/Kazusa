# Kazusa

*A research workspace for building a continuity-bearing AI system — not a chatbot wearing a clever hat.*

If you came here expecting “just add memory, vibes, and a persona prompt,” this repository will disappoint you efficiently. `Kazusa` is being designed as a system whose identity should emerge through structured experience, integration, bounded change, and relationship-grounded development.

## What This Is

- A core R&D project for continuity, appraisal, integration, relationship grounding, and behavioral safety.
- A workspace that treats identity as something developed over time, not instantly patched in by a dramatic single turn.
- A place for architecture, specifications, evaluation, prototypes, and governance documents.

## What This Is Not

- Not a fixed persona prompt.
- Not a roleplay shell.
- Not a memory-enhanced chatbot with better packaging.
- Not a system whose identity should be rewritten because one conversation was unusually persuasive.

## Repository Map

- `CoreDevelopment/` — active R&D workspace for architecture, specifications, evaluation, prototypes, and core design.
- `RuntimeStewardship/` — later-stage runtime operations, observation, and stewardship.
- `discussion_archive.md` — high-level discussion log.
- `full_dialogue_archive.md` — fuller dialogue archive.

## Current Focus

Right now the priority is to establish the R&D foundation inside `CoreDevelopment/`, especially:

- a continuity kernel,
- an experience and appraisal model,
- an integration loop,
- a relationship-grounding model,
- a behavioral safety envelope,
- and an evaluation harness for continuity, drift, and sycophancy.

## Working Bias

The project is deliberately opinionated:

- continuity over immediacy,
- integration over imitation,
- relationship without submission,
- growth without drift,
- safety at the behavior layer.

In short: less “one prompt to rule them all,” more “careful architecture with receipts.”
