# Kazusa

*一个用于构建“具连续性的 AI 系统”的研发工作区——不是给聊天机器人戴顶聪明帽子。*

如果你以为这里是“加点记忆、加点设定、再来一点玄学 prompt 就能起飞”，这个仓库会很礼貌地让你失望。`Kazusa` 想做的是另一件更严格、也更有意思的事：让系统的身份在结构化经验、整合过程、受限变化与关系性扎根中逐步形成。

## 这是什么

- 一个围绕连续性、经验评估、整合循环、关系扎根与行为安全展开的核心研发项目。
- 一个把“身份”视为长期发展结果，而不是单轮对话里临时热修补丁的工作区。
- 一个放架构、规范、评估、原型和治理文档的地方。

## 这不是什么

- 不是固定人格 prompt。
- 不是角色扮演外壳。
- 不是“包装更精致一点的记忆型聊天机器人”。
- 不是那种因为某次对话特别有说服力，就该当场改写身份的系统。

## 仓库结构

- `CoreDevelopment/` — 当前活跃的研发区，放架构、规范、评估、原型与核心设计。
- `RuntimeStewardship/` — 后续运行期的观察、运维与 stewarding 工作区。
- `discussion_archive.md` — 高层讨论记录。
- `full_dialogue_archive.md` — 更完整的对话归档。

## 当前重点

目前的优先级，是先在 `CoreDevelopment/` 里把研发地基打稳，重点包括：

- continuity kernel，
- experience and appraisal model，
- integration loop，
- relationship-grounding model，
- behavioral safety envelope，
- evaluation harness for continuity, drift, and sycophancy。

## 工作取向

这个项目的偏好很明确：

- 连续性优先于即时迎合，
- 整合优先于模仿，
- 关系不等于服从，
- 成长不能漂移失控，
- 安全主要落在行为层。

一句话总结：少一点“一条 prompt 定天下”，多一点“有追溯、有约束、能稳步成长的架构”。
